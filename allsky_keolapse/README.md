# Allsky Keolapse Module

Creates a timelapse video with the night's keogram wrapped as a ring around the
circular sky image, with a progress sweep showing the current position in the
night. Inspired by [indi-allsky](https://github.com/aaronwmorris/indi-allsky)'s
keogram timelapse feature.

Built for the Allsky **v2025** module architecture (`ALLSKYMODULEBASE`).

## How it works

After the night ends (the `nightday` event), the module reads the night's
captured images and the keogram generated by Allsky, wraps the keogram as a
ring around each frame, and encodes the result with ffmpeg to
`<images>/<date>/keolapse/keolapse-<date>.mp4`.

## Settings pulled from core Allsky

- Image extension is derived from the core `filename` setting.
- With **Use Timelapse Settings** enabled, the core `timelapsewidth`,
  `timelapseheight`, `timelapsefps`, `timelapsebitrate`, `timelapsevcodec`,
  `timelapsepixfmt` and `timelapsefflog` settings are used.
- Uploads use the standard `scripts/upload.sh` and honor `uselocalwebsite`,
  `useremotewebsite` and `useremoteserver`. The video is uploaded with its
  dated filename (`keolapse-YYYYMMDD.mp4`) into a dedicated subdirectory
  (default `keolapses`, configurable) so it never overwrites the regular
  timelapse video. For remote (sftp/scp/ftp) destinations this subdirectory
  must already exist on the server; the local-website subdirectory is created
  automatically.

## Events

- `nightday` — normal generation after the night ends.
- `periodic` — used only for the testing features (test-data generation,
  alignment/debug images). With testing disabled, periodic runs are skipped so
  a video is not regenerated every periodic cycle.

The module also supports the WebUI **[Test Module]** button; in test runs the
video is only uploaded if **Upload test video** is enabled.

## Overlay variables

On success the module publishes `AS_KEOLAPSE_VIDEO`, `AS_KEOLAPSE_DATE`,
`AS_KEOLAPSE_FRAMES` and `AS_KEOLAPSE_DURATION` for use in overlays.

## Dependencies

Uses OpenCV, NumPy and ffmpeg, all of which are installed by core Allsky. No
extra requirements.

### Thumbnails
- A JPEG thumbnail is generated from the video and uploaded to a `thumbnails`
  subdirectory of the upload directory (e.g. `keolapses/thumbnails/keolapse-YYYYMMDD.jpg`),
  matching the Allsky website gallery convention. Controlled by the
  "Create & Upload Thumbnail" setting (default on) and "Thumbnail Width"
  (default 100). For remote destinations the `thumbnails` directory must
  already exist on the server.
